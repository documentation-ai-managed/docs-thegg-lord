## Overview

Connect Thegg Lord to popular third-party services like GitHub and Slack to streamline your documentation workflow. Use webhooks for real-time notifications, leverage the API for custom extensions, and embed external content seamlessly. These integrations help you automate updates, sync content, and extend functionality without leaving your docs.

<Callout kind="info">
  All integrations use secure HTTPS connections and support OAuth 2.0 where applicable.
</Callout>

## Setting Up Webhooks

Webhooks notify external services about events like page updates or new comments. Follow these steps to configure them.

<Steps>
  <Step title="Navigate to Settings" icon="settings">
    Go to your project dashboard at `https://dashboard.example.com/projects/{projectId}/settings/webhooks`.
  </Step>
  <Step title="Add Webhook URL" icon="plus">
    Enter your endpoint URL, e.g., `https://your-webhook-url.com/thegglord`.
  </Step>
  <Step title="Select Events" icon="list">
    Choose events like `page.updated` or `comment.created`.
  </Step>
  <Step title="Test and Save" icon="check-circle">
    Send a test payload and save the webhook.
  </Step>
</Steps>

Here is a sample webhook payload:

<CodeGroup tabs="JavaScript,Python">
  ```javascript
  {
    "event": "page.updated",
    "projectId": "proj_123abc",
    "pageId": "page_xyz789",
    "timestamp": "2024-01-15T10:30:00Z",
    "user": {
      "id": "user_456def",
      "email": "user@example.com"
    }
  }
  ```
  ```python
  {
    "event": "page.updated",
    "projectId": "proj_123abc",

      "id": "user_456def",

  ```
</CodeGroup>

## API Access for Custom Extensions

Build custom integrations using the REST API at `https://api.example.com/v1`.

<ParamField path="projectId" param-type="string" required="true">
  Your project identifier.
</ParamField>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer `{YOUR_API_KEY}`.
</ParamField>

Send a request to list integrations:

<Request tabs="cURL,JavaScript">
  ```bash
  curl -X GET https://api.example.com/v1/projects/{projectId}/integrations \
    -H "Authorization: Bearer YOUR_API_KEY"
  ```
  ```javascript
  const response = await fetch('https://api.example.com/v1/projects/{projectId}/integrations', {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY'
    }
  });
  const data = await response.json();
  ```
</Request>

<Response tabs="200">
  ```json
  {
    "integrations": [
      {
        "id": "int_001",
        "type": "github",
        "enabled": true,
        "config": {}
      }
    ]
  }
  ```
</Response>

## Best Practices for Secure Connections

Follow these guidelines to keep integrations safe.

<Expandable title="Security Checklist" default-open="true">
  - Rotate API keys regularly via dashboard.
  - Use HTTPS-only endpoints for webhooks.
  - Validate payloads with HMAC signatures.
  - Limit scopes in OAuth apps to read-only where possible.
  - Monitor webhook delivery logs for failures.
</Expandable>

<Callout kind="alert">
  Never expose `YOUR_API_KEY` in client-side code or public repositories.
</Callout>
