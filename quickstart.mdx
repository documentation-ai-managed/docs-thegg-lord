## Prerequisites

<Callout kind="info">

Before starting, ensure you have:
- A modern web browser like Chrome or Firefox
- An email address for account verification
- Optional: Git installed for repository imports

</Callout>

## Create Your Account

Follow these steps to sign up and verify your Thegg Lord account.

<Steps>
  <Step title="Visit the Sign-Up Page" icon="user-plus">

    Navigate to `https://dashboard.example.com/signup` and enter your email and a strong password.

  </Step>
  <Step title="Verify Your Email" icon="mail">

    Check your inbox for a verification email from Thegg Lord. Click the link to activate your account.

  </Step>
  <Step title="Complete Onboarding" icon="check-circle">

    Log in and complete the quick onboarding quiz to set your preferences for documentation styles.

  </Step>
</Steps>

## Set Up Your Default Workspace

After logging in, create your first documentation workspace.

<Steps>
  <Step title="Create New Workspace" icon="plus">

    From the dashboard, click **New Workspace** and name it `MyProjectDocs`.

  </Step>
  <Step title="Configure Workspace Settings" icon="settings">

    Set the brand color to `#3B82F6` and enable features like version history.

  </Step>
  <Step title="Save and Publish" icon="save">

    Save changes. Your workspace is now live at `https://docs.example.com/myprojectdocs`.

  </Step>
</Steps>

## Import Initial Project Files

Import your existing documentation using one of these methods.

<Tabs>
  <Tab title="Git Repository" icon="git-branch">

    Connect your Git repo for automatic syncing.

    <Steps>
      <Step title="Add Repository URL">

        Enter `https://github.com/yourusername/myproject.git`.

      </Step>
      <Step title="Authenticate">

        Provide your GitHub token: `YOUR_GITHUB_TOKEN`.

      </Step>
    </Steps>

  </Tab>
  <Tab title="Manual Upload" icon="upload">

    Drag and drop files directly.

````markdown
Upload these files:
- README.md
- docs/introduction.mdx
- api-reference.json
````

  </Tab>
  <Tab title="CLI Import" icon="terminal">

    Use the Thegg Lord CLI for bulk imports.

````bash
npx thegglord import --workspace MyProjectDocs --path ./docs
````

  </Tab>
</Tabs>

## Basic Project Organization Tips

Organize your projects for better maintainability.

<CodeGroup tabs="Structure,Config">
```markdown
# Recommended folder structure
docs/
├── introduction.mdx
├── quickstart.mdx
├── api/
│   └── endpoints.mdx
└── changelog.mdx
```
```json
{
  "workspace": "MyProjectDocs",
  "features": {
    "search": true,
    "analytics": true
  }
}
```
</CodeGroup>

<Callout kind="tip">

Use consistent naming like `quickstart.mdx` and group related files in folders such as `api/` or `guides/`.

</Callout>
